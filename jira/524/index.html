              Description

              I am using XStream to debug a hard to reproduce problem. The goal for me was to quickly dump an XML representation of the some of the internal state of the application.
              I attach my debugger and evaluate something like

              new XStream().toXML( myObject );

              Unfortunately this causes a StackOverflowError while converting the object. Because of my particular use case I am not able to give you a test case to reproduce the problem.

              I am also not able to debug the xstream evaluation while debugging the state of the program. I could maybe achieve it by placing the xstream dumping code inside the program being debugged.
